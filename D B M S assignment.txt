Question 3. What is Difference between DBMS and RDBMS?

Answer 3.
       > DBMS (Database Management System): A software used to store and manage data, but it does not require relational tables.
Stores data in files or non-relational form No relationship between data Does not support normalization	Suitable for small amount of data	
Example: File system, XML database, dBase

	>RDBMS (Relational Database Management System): A type of DBMS that stores data in relational (table) format and uses keys to maintain relationships.

Stores data in tables (rows & columns)Tables are related using keys (Primary Key, Foreign Key)Supports normalization Suitable for large databases
Example: MySQL, Oracle, SQL Server, PostgreSQL

Question 4. What is MF Cod Rule of RDBMS Systems?
Answer 4. there are total 13 rules....

> Rule 1 – Information Rule
All data must be stored in tables (rows & columns).

> Rule 2 – Guaranteed Access Rule
Each value can be accessed using table name, column name, and primary key.

> Rule 3 – Systematic Null Handling
NULL values must be handled properly (unknown / missing).

> Rule 4 – Dynamic Online Catalog
Database dictionary should also be stored as tables.

> Rule 5 – Comprehensive Data Sublanguage Rule
The system must support at least one powerful language (like SQL).

> Rule 6 – View Updating Rule
Views should be updatable whenever possible.

> Rule 7 – High-Level Insert, Update, Delete
SQL must support operations on sets of records, not only one row at a time.

> Rule 8 – Physical Data Independence
Changes in storage (files, hardware) should not affect data.

> Rule 9 – Logical Data Independence
Changes in table structure should not affect applications.

> Rule 10 – Integrity Independence
Integrity rules (PK, FK, constraints) should be stored separately and managed by the database.

> Rule 11 – Distribution Independence
Data can be distributed across multiple locations without affecting users.

> Rule 12 – Non-subversion Rule
If low-level access exists, it must not bypass security or constraint

Question 6.What is DDL Interpreter?
Answer 6. DDL Interpreter is the part of a DBMS that reads and executes DDL (Data Definition Language) commands like CREATE, ALTER, DROP, and modifies the database
 structure.

> DDL commands (CREATE, ALTER, DROP, TRUNCATE).
> Checks syntax of the command.
> Changes the database schema (tables, indexes, constraints).
> Updates the data dictionary (metadata stored by the DBMS).

Question 9.What is save Point? How to create a save Point write a Query?10.What is trigger and how to create a Trigger in SQL?
Answer 9. A Savepoint is a point inside a transaction where you can pause or mark the current state of the database so that you can roll back only part of the 
transaction instead of the whole transaction.

 START TRANSACTION;
 INSERT INTO student VALUES (101, 'Amit', 85);
 SAVEPOINT sp1;
 INSERT INTO student VALUES (102, 'Rahul', 90);
 ROLLBACK TO sp1;   -- This will undo second INSERT only
 COMMIT;

Question 10. What is trigger and how to create a Trigger in SQL?
Answer 10. A Trigger is a stored program in SQL that runs automatically when a specific operation (INSERT, UPDATE, DELETE) happens on a table. 

 Types of Triggers

 > BEFORE Trigger
 > AFTER Trigger

CREATE TABLE student_log (
    id INT,
    action VARCHAR(20),
    log_time TIMESTAMP
);

trigger creation

CREATE TRIGGER after_student_insert
AFTER INSERT ON student
FOR EACH ROW
INSERT INTO student_log (id, action, log_time)
VALUES (NEW.rollno, 'INSERT', NOW());
